Módulo 1. Bases de datos

Objetivo

Crear una base de datos para relacionar los vuelos con los pasajeros del Aeropuerto Borregos.

Nombre

Aeropuerto borregos
El cliente solicitó ser claros y concisos con los nombres tanto de las tablas como de sus atributos.

Tablas

  1. Vuelo
  2. Pasajero
Se utilizaron dos tablas, una para almacenar los datos de los vuelos ofrecidos y otra para la de los pasajeros en el Aeropuerto Borregos. La primera contiene las características que comparten todos los pasajeros que abordan tal vuelo como el modelo del avión, puerta de abordaje, tiempo y terminal de partida y llegada. Por otra parte, en la tabla de pasajero se almacenan los datos únicos de cada pasajero, es decir, lo que puede personalizar de su experiencia de vuelo de acuerdo a sus necesidades como la clase, si es que tendrá retorno, el número de su asiento y el id del vuelo mediante el cual viajarán.

Dato para relacionar vuelo-pasajero: idvuelo

Buscamos un dato en común entre el pasajero y el vuelo para relacionar las tablas, y este fue el identificador del vuelo (idvuelo), pues, es el vuelo que abordará el pasajero. idvuelo es llave primaria en vuelo y foránea en pasajero

vuelo
Atributo Tipo de Dato Justificación
idvuelo MEDIUMINT(9) Este dato es de tipo MEDIUMINT(9) y es autoincremental, es decir, va aumentando de manera automática cada que se agregue un vuelo nuevo. Nos sirve para identificar los vuelos. Este campo es NOT NULL, requiere que sea llenado con un valor. Esta es la primary key de la tabla.
modelavion VARCHAR(4) Este atributo consiste de una combinación de una letra mayúscula y tres números enteros, por lo que, se utilizó una cadena VARCHAR(4) para permitir tener la combinación de letras con números y limitarla a una extensión de cuatro caracteres. Este nos indica el modelo del avión que realizará el vuelo en cuestión. Este campo es NOT NULL, requiere que sea llenado con un valor.
salida DATETIME El atributo es de tipo DATETIME, lo que guarda la fecha y hora en la que el vuelo sale del aeropuerto. Se registra en el formato YYYY-MM-DD hh:mm:ss. Se utilizó DATETIME en vez de TIMESTAMP ya que el primero almacena un valor específico definido para la fecha y hora; en cambio, TIMESTAMP almacena el tiempo actual en el que se modifica el record. Este campo es NOT NULL, requiere que sea llenado con un valor.
llegada DATETIME El atributo es de tipo DATETIME, lo que guarda la fecha y hora en la que el vuelo llega a su destino. Se registra en el formato YYYY-MM-DD hh:mm:ss. Se utilizó DATETIME sobre TIMESTAMP ya que el primero almacena un valor específico definido para la fecha y hora; en cambio, TIMESTAMP almacena el tiempo actual en el que se modificó el record. Este campo es NOT NULL, requiere que sea llenado con un valor.
aerolinea VARCHAR(20) La aerolínea es un atributo de tipo VARCHAR(20), permitiendo guardar una cadena de caracteres (en este caso el nombre de las aerolíneas) de una longitud máxima de 20. Este campo es NOT NULL, requiere que sea llenado con un valor.
terpartida TINYINT Este atributo es un TINYINT, ya que guardará el número de la terminal de la que partirá el avión y este no será mayor a 127. Este campo es NOT NULL, requiere que sea llenado con un valor.
terllegada TINYINT Este atributo es un TINYINT, ya que guardará el número de la terminal de llegada del avión y este no será mayor a 127 (Maximum value signed). Este campo es NOT NULL, requiere que sea llenado con un valor.
retraso BOOL Es de tipo BOOL, ya que solamente guardará un 1 o un 0, indicando si el vuelo viene retrasado o no, respectivamente. Este campo es NOT NULL, requiere que sea llenado con un valor.
puerta TINYINT La puerta es de tipo TINYINT, ya que este número no superará 127 (Maximum value signed). Este atributo es la puerta de abordaje. Este campo es NOT NULL, requiere que sea llenado con un valor.
pasajero
idvuelo MEDIUMINT Es un MEDIUMINT, nos sirve para identificar en qué vuelo estará el pasajero. Este campo es NOT NULL, por lo tanto siempre se asignará un valor al atributo. Esta es la foreign key que se utiliza
nombrepasajero VARCHAR(30) Es un VARCHAR(30), ya que la longitud de los nombres de los pasajeros puede variar y su propósito es saber el nombre de las personas que abordan los vuelos. Este campo es NOT NULL, por lo tanto siempre se asignará un valor al atributo.
clase VARCHAR(3) Se utilizara una abreviatura para la clase de asiento, esto indicará en qué sección del avión se sentará el pasajero. Este campo es NOT NULL, por lo tanto siempre se asignará un valor al atributo.
retorno BOOL ó TINYINT(1) Es un BOOL, ya que solo queremos saber si el pasajero regresará o no.Este campo es NOT NULL, por lo tanto siempre se asignará un valor al atributo.
asiento VARCHAR(3) se utilizaran solo 3 datos, dos números para indicar la fila y la letra para el asiento de dicho pasajero. Este campo es NOT NULL, por lo tanto siempre se asignará un valor al atributo.
